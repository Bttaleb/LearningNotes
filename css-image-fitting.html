Why Your Images Are Cut Off

The 3 CSS properties causing the crop — and how to fix them

The Problem: Your Current CSS

Here's what you have right now in style.css (lines 135–141):

.image-gallery img { width: 280px; /* forces every image to exactly 280px wide */ height: 200px; /* forces every image to exactly 200px tall */ object-fit: cover; /* scales image UP to fill the box, CROPS the rest */ border-radius: 8px; border: 1px solid #30363d; }

Three properties are working together to create the crop:

1 Fixed width: 280px

Forces the image container to be exactly 280px wide — regardless of the photo's natural width. If the photo is 800px wide, it gets squeezed.

2 Fixed height: 200px

Forces the container to be exactly 200px tall. A tall portrait photo (say 600px tall) now has to cram into 200px. Something has to give.

So what does object-fit: cover do?
It's the tiebreaker. When the image doesn't match the box dimensions, cover says: "Scale the image so it completely fills the box, then crop anything that overflows." That's why your photos look squared and cut off — the tops, bottoms, or sides are being sliced away.

Visual: What object-fit Actually Does

Think of it like framing a poster that's too big for the frame:

Value Behavior Result
cover Scale to fill the entire box, crop the overflow Image fills box but parts are hidden
contain Scale to fit entirely inside the box, add empty space if needed Full image visible, may have letterboxing
fill Stretch/squish to exactly match box dimensions Image distorted (squashed or stretched)
none Ignore the box, show image at its natural size May overflow or be way too big

The Fix

Replace lines 135–141 in your style.css with:

.image-gallery img { max-width: 48%; /* each image takes up ~half the gallery */ height: auto; /* let the image decide its own height */ /* no object-fit needed — natural aspect ratio is preserved */ border-radius: 8px; border: 1px solid #30363d; }
Why this works:
max-width: 48% Instead of a rigid pixel width, each image fills about half the gallery row — responsive and flexible.

height: auto This is the key line. It tells the browser: "calculate the height from the image's natural aspect ratio." No more forced 200px box = no more cropping.

No object-fit When width and height respect the image's natural ratio, you don't need object-fit at all. The image just... fits.

Quick Reference: When to Use What

You Want Use This
Full image visible, natural proportions max-width: 48%; height: auto;
Uniform grid (all same size), OK with cropping width: 280px; height: 200px; object-fit: cover;
Uniform grid, no cropping (letterbox OK) width: 280px; height: 200px; object-fit: contain;
Full-width hero image width: 100%; height: auto;
