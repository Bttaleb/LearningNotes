# C Programming ‚Äî Map of Content

Top-level dashboard for everything in this vault. Open the **graph view** (Cmd+G) to see how concepts connect.

## How this vault is organized

- **01 - Chapters/** ‚Äî One overview note per chapter. Acts as a hub linking to atomic concept notes.
- **02 - Concepts/** ‚Äî Atomic notes, one idea each. The connective tissue. Heavily cross-linked.
- **03 - Exercises/** ‚Äî Practice problems with my solutions and what I got wrong.
- **04 - Projects/** ‚Äî Small C programs I write to apply ideas.
- **05 - Snippets/** ‚Äî Quick-reference cheat sheets (format specifiers, escape sequences, etc).
- **_Templates/** ‚Äî Templates for new notes. Set Obsidian's Templates plugin to point here.

## Tag conventions

Use these consistently in frontmatter so search and graph views stay clean:

- `chapter/3` ‚Äî which chapter the note belongs to
- `topic/io`, `topic/pointers`, `topic/memory`, `topic/types`, etc.
- `status/stub`, `status/learning`, `status/review`, `status/mastered`
- `kind/concept`, `kind/exercise`, `kind/project`, `kind/snippet`

## Workflow

1. Reading a new section ‚Üí open the chapter file, add bullets to "Key ideas".
2. Hit a concept that deserves its own note ‚Üí `[[Concept Name]]` and Cmd+click to create it from template.
3. Want to test understanding ‚Üí make an entry in **03 - Exercises/**.
4. Want to apply it ‚Üí make a folder in **04 - Projects/** with a `.c` file and a notes `.md`.
5. Weekly: open the graph view, look for orphan notes, link them.
